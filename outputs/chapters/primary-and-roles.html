一次系统与设备角色

从一次结构定位功能，再理解不同 IED 在站内承担的职责

两种视角：一次设备与自动化功能

Substation 描述一次系统组织；IED 描述自动化设备的数据与服务。二者不是重复建模：一次设备例如断路器、隔离开关、互感器属于电气结构；逻辑节点例如 XCBR、MMXU、PTOC 描述与它们相关的自动化功能。LNode 是从一次位置引用 IED 内逻辑节点的桥梁。

Substation → VoltageLevel → Bay → ConductingEquipment / PowerTransformer / Function / LNode；LNode ── iedName、ldInst、prefix、lnClass、lnInst ──▶ IED 内的 LN 或 LN0。

一次系统层级速查

元素标准语义工程阅读要点
Substation厂站或变电站的一次系统顶层容器。可含电压等级、功能、设备等结构；名称是工程标识，不是跨项目固定枚举。
VoltageLevel电压等级容器。常包含 Bay；具体电压数值和命名应以工程设计为准。
Bay间隔，例如线路、主变、母联或电容器间隔。是一次组织单元，不是逻辑节点，也不等同于一台 IED。
ConductingEquipment导电设备的通用建模元素。设备类别由属性和工程结构体现；不要只凭 name 推断其一次功能。
PowerTransformer / TransformerWinding电力变压器及绕组的一次结构建模。可与电压等级、端子、功能和 IED 逻辑节点关联阅读。
Function一次系统中可组织功能的结构元素。用于表达系统功能结构，具体采用深度由项目工程规范决定。
LNode一次结构中的逻辑节点引用。使用 IED、LD、LN 类和实例属性，把一次位置关联到 IED 数据模型。

如何追踪一次设备到数据模型

  1. 从 Substation/VoltageLevel/Bay 找到目标间隔和一次设备。
  2. 查看该层级的 LNode，记录 iedName、ldInst、prefix、lnClass、lnInst。
  3. 在对应 IED 的 Server/LDevice/LN0|LN 中按这些标识定位逻辑节点。
  4. 再查看该 LN 的数据对象、DataSet、Inputs 和控制块，追踪测量、控制、GOOSE 或 SV 的工程关系。
常见误解：LNode 是引用而非 IED 内的数据实例；它能建立关联，却不替代 LN 本体的模型定义。

设备角色百科

角色通常职责在 SCD 中优先核对
保护装置基于测量与逻辑执行保护判据、跳闸与告警；可收发 GOOSE、订阅 SV。PTOC、PDIS、PTRC 等 LN；Inputs/ExtRef、数据集和 GOOSE。
测控装置遥测、遥信、遥控及间隔自动化。MMXU、XCBR、CSWI、CILO；报告与控制模型。
MU（合并单元）将 CT/VT 或电子式互感器量数字化，向过程层发布 SV/SMV；部分装置也接收 GOOSE。SampledValueControl、DataSet、Communication/SMV，以及 TCTR/TVTR 等模型。
智能终端面向一次设备机构的开入开出、位置、跳闸与联锁执行。XCBR、CSWI、GGIO，以及 GOOSE 输入输出。
网关 / 远动机汇集站内数据并与调度或其他外部系统交换。MMS/报告服务及工程私有扩展；外部规约映射通常不是 IEC 通用 SCL 语义。
以太网交换机承载站控层或过程层网络流量，可能负责 VLAN、优先级、冗余网络转发。不必然以完整 IED 数据模型出现；端口、转发表等常依赖厂商工具或私有扩展。
时间同步设备为站内装置提供统一时间基准，例如 PTP 或 SNTP/NTP 场景。核对项目时间同步方案与 IED 支持能力；不从单个 SCL 标签推断准确度或全网同步状态。
角色不是标准 XML 枚举。“保护装置”“MU”“智能终端”等通常通过 IED 内的 LN、通信服务、实际接线和设备资料综合识别；IED@type 或名称可辅助判断，但工程命名并不统一。

一次系统排查清单

  1. 一次设备名称、类型和端子连接是否与图纸和间隔结构一致。
  2. 每个关键一次功能是否有合理的 LNode 到 IED/LN 映射。
  3. 断路器控制、闭锁、保护跳闸与位置反馈是否跨一次、IED、GOOSE 三个视角一致。
  4. MU 的采样值发布方、接收保护/测控和过程层网络地址是否形成闭环。
  5. 不要将厂商私有元素或 IED 名称直接当作标准设备角色定义。
