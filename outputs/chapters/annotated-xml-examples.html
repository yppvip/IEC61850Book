注释式 XML 示例库

原创、最小、可读的 SCL 片段：读出标准语义，再落到工程配置检查

使用边界

所有名称、地址、数值和结构均为原创教学示例，故意省略无关必需段落，不能直接作为 ICD、SCD 或 CID 导入/下装。每例分别标出标准语义、工程配置项和常见错误；项目中仍须以设备一致性声明、工具支持和项目规范为准。

1. IED / LDevice / LN：功能模型的住址

<IED name="IED_BAY01">
  <AccessPoint name="AP1"><Server>
    <LDevice inst="LD0">
      <LN0 lnClass="LLN0" inst=""/>
      <LN lnClass="XCBR" inst="1" lnType="XCBR_TYPE_01"/>
    </LDevice>
  </Server></AccessPoint>
</IED>
注释说明
标准语义IED → AccessPoint → Server → LDevice → LN0/LN 是数据模型层级。XCBR 是断路器相关功能 LN 类；inst 用于区分同类实例。
工程配置项IED@name、AccessPoint@name、LDevice@inst、LN@prefix/lnClass/inst/lnType 是跨引用和类型追溯的关键标识。
常见错误把 XCBR 直接等同于一次断路器；或只按 lnClass 查找、忽略 prefix 与 inst，导致定位到错误 LN。

2. DataSet / FCDA：声明“要选哪些数据”

<LN0 lnClass="LLN0" inst="">
  <DataSet name="dsBreaker">
    <FCDA ldInst="LD0" lnClass="XCBR" lnInst="1"
          doName="Pos" daName="stVal" fc="ST"/>
  </DataSet>
</LN0>
注释说明
标准语义DataSet 是数据引用清单；此 FCDA 定位到 LD0 内 XCBR1 的 Pos.stVal，功能约束为 ST。
工程配置项name 供控制块引用；FCDA 的 LD、LN、DO、DA、FC 必须能解析到发送 IED 的实际模型。
常见错误将 DataSet 当成实时数值容器；漏写或错写 fc；只凭 doName 相同就认为是同一数据。

3. GOOSE 发布与订阅：控制块、网络、ExtRef 三段闭环

<!-- 发送 IED 的 LN0 -->
<GSEControl name="gcbTrip" datSet="dsTrip" appID="TRIP01" confRev="1"/>

<!-- Communication 中的发送网络配置 -->
<GSE ldInst="LD0" cbName="gcbTrip">
  <Address><P type="MAC-Address">01-0C-CD-01-00-01</P>
    <P type="APPID">0001</P><P type="VLAN-ID">001</P></Address>
</GSE>

<!-- 接收 LN 的输入引用（属性按工程工具生成） -->
<Inputs><ExtRef iedName="IED_BAY01" ldInst="LD0" lnClass="PTRC"
  doName="Tr" daName="general" serviceType="GOOSE"/></Inputs>
注释说明
标准语义GSEControl@datSet 指定发送内容；Communication/GSE 提供网络侧关联；接收 LN 的 ExtRef 表达外部输入绑定。
工程配置项发送 DataSet/FCDA、cbName、LD 实例、MAC、APPID、VLAN 以及接收侧引用须整体一致；具体 ExtRef 属性以工具和 SCL 版次为准。
常见错误只核对 MAC 而未核对控制块和数据集；把 confRev 当作网络时延；或认为 ExtRef 存在就已保证现场订阅生效。

4. MU → 保护 SV：采样值发布链

<!-- MU 发送侧 LN0 -->
<SampledValueControl name="smvMu01" datSet="dsCurrent" smvID="MU01_CURRENT"/>

<!-- Communication 中的网络侧条目 -->
<SMV ldInst="LD0" cbName="smvMu01">
  <Address><P type="MAC-Address">01-0C-CD-04-00-01</P>
    <P type="APPID">4001</P><P type="VLAN-ID">002</P></Address>
</SMV>

<!-- 保护 LN 的输入引用 -->
<Inputs><ExtRef iedName="MU01" ldInst="LD0" lnClass="TCTR"
  doName="Amp" daName="instMag" serviceType="SMV"/></Inputs>
注释说明
标准语义SV/SMV 面向连续采样量。发送控制块引用 DataSet，通信侧 SMV 关联网络，接收保护模型以输入引用绑定来源。
工程配置项采样数据集的通道/相序、smvID、LD 与控制块名、MAC/APPID/VLAN、采样率和接收配置必须按项目设计闭环。
常见错误把 SV 当 GOOSE 使用；只比对网络地址而未检查通道顺序、采样参数、互感器量纲或接收 IED 的实际订阅。

5. ReportControl：面向客户端的报告

<LN0 lnClass="LLN0" inst="">
  <DataSet name="dsMms">
    <FCDA ldInst="LD0" lnClass="MMXU" lnInst="1"
          doName="A" daName="mag" fc="MX"/>
  </DataSet>
  <ReportControl name="rptMeasure" datSet="dsMms" buffered="true">
    <TrgOps dchg="true" qchg="true" period="true"/>
    <OptFields seqNum="true" timeStamp="true" dataSet="true"/>
  </ReportControl>
</LN0>
注释说明
标准语义ReportControl 定义报告控制块并引用 DataSet；TrgOps 描述触发类别，OptFields 选择报告附带字段。
工程配置项是否缓冲、触发项、可选字段、允许客户端数量和使能方式均受 IED 能力及客户端工程配置约束。
常见错误把 Report 当作 GOOSE 多播；只因 buffered="true" 就假设历史一定完整；忽略客户端是否已正确使能对应 RCB。

6. 控制对象与 ctlModel：模型属性不是操作许可

<DOI name="Pos">
  <DAI name="ctlModel"><Val>sbo-with-enhanced-security</Val></DAI>
</DOI>

<!-- 控制相关数据集/服务仍须结合实际 LN、DOType 与 IED 能力理解 -->
注释说明
标准语义ctlModel 是控制对象相关的数据属性值，示例表达先选择后执行的控制模型语义；具体对象结构须沿类型模板确认。
工程配置项控制对象、Oper/SBO 服务能力、CSWI/CILO、权限、远方/就地、闭锁及测试策略都应按项目配置和试验验证。
常见错误将 DAI/Val 直接当作当前运行状态；或仅看到 ctlModel 就认定遥控可执行，忽略闭锁、权限、模式及现场安全条件。

如何把示例用于实际 SCD

  1. 将示例中的占位名称替换为真实工程中的 IED、LD、LN、DataSet 和控制块名称，但不要复制示例地址或数值。
  2. 按“实例 → 类型模板 → 通信网络 → 接收引用”的顺序追踪，避免只在一个 XML 层级内判断。
  3. 把 SCL 结构校验、引用一致性校验、IED 下装和现场功能验收分别记录；前一项通过不代表后一项已通过。
