类型与标准模型索引

逻辑节点、公共数据类、类型模板和版本差异的查询入口

如何使用标准模型索引

IEC 61850 通过逻辑节点（LN）、数据对象（DO）、公共数据类（CDC）和数据属性（DA）组织语义。LN 类说明“功能是什么”，CDC 说明“这类数据有哪些典型属性结构”，具体 SCL 文件中的实例再通过类型模板与 lnType、DO@type、DA@type 关联。

本索引的定位：按功能组提供高频标准模型的原创导读与检索别名。它不是标准原文的逐条重印；完整模型与设备实际支持项仍应以相应标准、ICD 和一致性声明核对。

类型模板关系

LN/LN0@lnType → LNodeType（DO 列表）→ DO@type → DOType（DA/SDO 结构）→ DA@type → DAType（BDA 结构）；DA@bType="Enum" 常关联 EnumType（枚举编码与文字）。
模板回答的问题典型误区
LNodeType某类 LN 包含哪些 DO？同一 lnClass 的不同 lnType 可能具有不同工程模型扩展或可选项。
DOType某个 DO 内含哪些 DA/SDO？不要将 DO 名称直接当成全部属性路径。
DAType结构化 DA 内含哪些 BDA？复合属性的成员应继续展开，不能只查看外层名称。
EnumType枚举文字对应何种编码？同一显示文字在不同枚举类型中不必具有相同数值或语义。

逻辑节点功能组速查

功能组示例 LN 类通常含义
公共与物理设备LLN0、LPHD逻辑设备公共行为、物理设备健康/标识等。
通用 I/O 与调度GGIO、GSAL通用输入输出、告警等；特别是 GGIO 必须结合 DO 名和描述理解。
测量与计量MMXU、MSQI、MMTR三相测量、对称分量、计量等。
一次设备模型XCBR、XSWI、TCTR、TVTR、YPSH断路器、开关、互感器、一次设备相关功能。
控制与闭锁CSWI、CILO、CALH开关控制、闭锁和告警处理等。
保护PTOC、PDIS、PDIF、PTRC、RREC过流、距离、差动、跳闸条件、重合闸等保护/自动化功能。
电能质量QFVR、QVVR、QITR频率/电压变化、谐波等电能质量相关功能。
分布式能源与电网自动化DRCC、DRCS、FLSH 等特定应用域模型；是否出现在变电站 SCD 取决于项目范围。
LN 类不等于完整功能结论。例如看到 PTOC 可知存在过流保护相关模型，但整定策略、保护配合、投退条件和动作出口仍须查看具体 DO/DA、设置、逻辑及工程图纸。

常见公共数据类（CDC）

CDC常见数据语义阅读提示
SPS / DPS单点状态 / 双点状态通常结合 stVal、q、t；双点状态的具体编码须看类型定义。
INS / ENS整数状态 / 枚举状态枚举状态应沿 EnumType 确认文字和编码。
ACT / ACD保护动作信息 / 保护动作方向等复合信息按相别或方向等成员理解，不宜简化为单一布尔值。
MV / CMV / SAV测量值 / 复数测量值 / 采样模拟值需同时检查数值、质量、时标、单位、倍率和采样上下文。
WYE / DEL星形量 / 三角形量等多相结构成员布局与相序的工程含义需结合类型和项目约定。
SPC / DPC / APC单点、双点、模拟控制应连同 ctlModel、Oper/SBO、闭锁和权限阅读。
INC / ENC / ING整数、枚举、整数设置区分 FC 与当前/编辑/设置组值。
ASG / CSG / VSG模拟、计数器、向量设置组属于设置语义，不能直接当作运行测量值。

Edition 2 / 2.1 提示

Edition 2.1 对模型和工程交换持续扩展。某个 LN、CDC、SCL 元素或属性是否可以使用，应同时确认：目标 IED 的 ICD/一致性声明、工程工具支持范围、项目版次要求以及实际 XML 命名空间和实例结构。不要仅以“文件声明 Ed.2.1”推断设备支持所有新增模型。

扩充路线

  1. 先按 LN 功能组建立每个类的中文解释、典型 DO、常见工程角色和相关 CDC。
  2. 再按 CDC 补齐 DA 结构、功能约束、质量/时标和控制/设置差异。
  3. 将实例中出现的 lnClass、lnType 和 EnumType 与索引对应，但私有类型保持“扩展”标记。
  4. 对 Edition 差异、可选项和厂商支持情况建立单独的版本/能力提示，不把它们混写为标准必选项。
