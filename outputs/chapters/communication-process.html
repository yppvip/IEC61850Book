通信与过程层

网络接入、MMS 报告、GOOSE、采样值与时间同步的工程关系

Communication 的网络层级

Communication → SubNetwork → ConnectedAP → Address；ConnectedAP 下的 GSE / SMV 将 IED 内控制块关联到网络地址。IED 内定义“发布什么”，Communication 定义“怎样接入并在何处发布”。
元素作用核对重点
SubNetwork通信子网，例如站控层或过程层网络。name、type 是工程配置；网络名称并非统一标准命名。
ConnectedAPIED 某个 AccessPoint 接入某通信子网。iedName、apName 要能与 IED/AccessPoint 对应。
Address/P承载网络地址参数。查看 P@type 的语义，例如 IP、IP-SUBNET、IP-GATEWAY、MAC-Address、APPID、VLAN-ID、VLAN-PRIORITY。

MMS 与报告

MMS 是 IEC 61850 常用的客户端/服务器服务基础。报告服务由 ReportControl 定义报告控制块，通常引用 DataSet；客户端读取、使能或接收报告的具体行为仍受 IED 服务能力与工程配置约束。

对象/属性工程含义
ReportControl@datSet指定报告使用的数据集。
buffered区分缓冲报告与非缓冲报告的配置语义；是否可用及容量依 IED 能力。
TrgOps报告触发选项，例如数据变化、品质变化、周期完整性等相关条件。
OptFields报告可选字段选择，决定报告中可携带的辅助信息类别。
RptEnabled与允许的报告实例/客户端相关的配置；不能仅凭其存在判断现场已使能。

GOOSE：快速事件与联锁

GOOSE（Generic Object Oriented Substation Event）面向状态和事件的快速多播，常用于跳闸、闭锁、位置、告警和联锁。发送侧的 GSEControl 引用 DataSet，通信侧的 GSE 为该控制块提供网络配置，接收侧通过 Inputs/ExtRef 表达工程订阅。

发送：LN0/GSEControl → DataSet/FCDA；网络：ConnectedAP/GSE → MAC、APPID、VLAN、MinTime、MaxTime；接收：LN/Inputs/ExtRef → 来源数据与服务关联。
MinTime/MaxTime 不是网络时延保证。它们与 GOOSE 重发节奏有关。保护动作时限、网络负荷和报文性能需依据项目规范、网络设计和现场测试验证。

SV / SMV：过程层采样值

SV（Sampled Values）是连续电流、电压等电气量的采样值传输服务；工程中常以 SMV 指代 SCL 中的采样值配置。典型路径是互感器量 → MU 采样/数字化 → 以太网 SV 多播 → 保护或测控订阅。

位置元素/信息阅读要点
IED 内SampledValueControl、datSet、smvID定义发布控制块及采样值数据集；可用属性随 SCL 版本与工具能力变化。
CommunicationConnectedAP/SMV将发布控制块关联到网络；核对 ldInst、cbName 和 Address。
接收 IEDInputs/ExtRef接收 LN 引用外部数据；须和发送侧数据集、控制块及地址整体核对。
SV 与 GOOSE 不同：SV 是高频连续采样量流；GOOSE 是状态/事件消息。二者都可能出现 MAC、VLAN、APPID，却有不同的业务语义和数据检查方法。

地址、VLAN、冗余与时间同步

项目含义与边界
MAC / APPID常用于识别二层多播服务流。应检查发送、接收和交换网络规划是否一致，不能将某个工程值当作通用默认值。
VLAN-ID / VLAN-PRIORITY用于二层隔离与优先级规划。具体数值来自项目网络设计。
IP 地址常服务于客户端/服务器通信等 IP 场景；IP、子网掩码、网关的含义取决于对应 P@type。
PRP / HSR 等冗余是网络冗余方案；是否采用及其端口、环网和监督配置要按项目网络设计与设备支持能力核对。
PTP / SNTP / NTP是常见授时方式。SCL 中的时间相关信息不等于已验证同步精度，需结合设备能力、时钟源与测试结果。

通信链路排查顺序

  1. 从接收 ExtRef 记录来源 IED、LD、LN、DO/DA、服务类型与关联控制块信息。
  2. 在发送 IED 定位对应 DataSet/FCDA 及 GSEControl 或 SampledValueControl。
  3. 到 Communication 中核对发送 ConnectedAP/GSE 或 SMV，以及 MAC、APPID、VLAN 等地址参数。
  4. 核对接收 IED 所在子网、设备接入点和网络设计，最后再检查交换机/VLAN/冗余配置。
  5. 报告问题按 IED 服务能力、ReportControl、数据集、客户端配置和 IP 网络路径依次定位。

需要从抓包继续定位时，进入 IEC 61850 报文解析，按 Ethernet/VLAN/TCP、APPID、BER TLV、GOOSE/SV/MMS 字段和 SCL 反查逐层确认。
