易混概念速查

用“对象是什么、在哪里、不能当成什么”快速区分 SCL 工程常见混淆

先记三条判断线

一次世界：设备、间隔、接线位置            → Substation / Bay / ConductingEquipment
信息模型：功能与数据                         → IED / LDevice / LN / DO / DA / FC
工程交换与通信：文件、发布、订阅、网络参数   → ICD / SCD / CID；GOOSE / SV / Report

同一个工程对象可在多个视角出现，但它们的“名字相近”不表示“类型相同”。下表优先解决阅读 SCD 时最常见的误判。

一次设备、LNode 与 LN

概念是什么通常在哪里不要当成
一次设备断路器、隔离开关、互感器、变压器等电气一次对象。Substation 下的 Bay、ConductingEquipment、PowerTransformer 等。不是 IED 内的数据模型，也不由某一个 LN 完整替代。
LNode从一次系统位置指向 IED 内逻辑节点的引用桥梁。一次系统层级，如 Bay、Function 等。不是 IED 内的 LN 实例，通常没有自身的 DO/DA 数据。
LN按自动化功能组织数据的逻辑节点，例如 XCBR、MMXU。IED/AccessPoint/Server/LDevice 内。不是物理设备本体；同一一次设备可关联多个 LN，一个 LN 也不等于一台设备。
阅读口诀：先在一次层定位“哪台设备/哪个间隔”，再经 LNode 追到 IED 中“哪个功能模型”。

IED、MU 与智能终端

概念核心职责在 SCD 中优先看边界
IED智能电子设备的通用工程模型。名称、AP、LD、LN、服务与通信接入。是模型和设备的通称；保护、测控、MU、智能终端都可能建模为 IED。
MU（合并单元）把电流、电压等采样数字化，典型地发布 SV/SMV。SampledValueControl、DataSet、Communication/SMV。是工程设备角色，不是 XML 元素名；不等于交换机或所有过程层装置。
智能终端面向一次机构处理开入开出、位置、跳闸及部分联锁执行。XCBR、CSWI、GGIO 与 GOOSE 输入输出。也是工程角色；实际职责、接口与命名必须结合图纸和厂商资料确认。

DO、DA 与 FC

概念回答的问题例子常见误解
DO这类功能数据是什么？位置、测量、控制等数据对象，如 Pos。DO 不是一个确定的可读数值；其内部结构由类型模板定义。
DA这个对象中的哪个属性？stVal、q、t、ctlModel。DA 不是 XML 属性；它是数据模型中的数据属性。
FC该数据属性按什么服务语义使用？ST 状态、MX 测量、CO 控制、SP 设置。FC 不是数据类型、不是实时值，也不是所有 XML 属性的通用枚举。

读一个数据引用时，至少同时记录 LN / DO / DA / FC；只看到 Pos 或 stVal 还不足以锁定完整语义。

GOOSE、SV 与 Report

服务主要传什么典型方向关键工程对象
GOOSE状态和事件，如跳闸、闭锁、位置、告警。发送 IED 多播，多个接收 IED 可订阅。GSEControl → DataSet → Communication/GSE → ExtRef
SV / SMV连续电流、电压等采样量。常由 MU 发布，保护/测控订阅。SampledValueControl → DataSet → Communication/SMV → ExtRef
Report按数据集和触发条件向客户端上送的数据。IED 与 MMS 客户端之间的客户端/服务器服务。ReportControl → DataSet → 客户端配置
相同的网络字段不代表相同服务。GOOSE 与 SV 都可能涉及 MAC、APPID、VLAN；Report 则通常与 MMS 客户端连接相关。不要用“都能传数据”替代服务语义和引用链检查。

ICD、CID 与 SCD

文件工程位置主要回答的问题不能据此断言
ICDIED 能力输入。该 IED 支持哪些数据模型、服务和可配置能力？不能说明它已完成本站工程配置或已在现场投运。
SCD全站系统集成配置。一次系统、IED、通信、类型模板和订阅关系如何汇集？不能单独证明每台 IED 已正确下装或现场功能已通过。
CID单台 IED 的已配置描述。面向该 IED 的配置交付通常是什么？具体导入/下装方式仍取决于设备和工程工具。

配置值、运行值与枚举编码

概念出现位置/来源正确理解不要混淆为
配置值例如 SCL 的 DOI/DAI/Val，或工程属性与设置相关信息。表达工程配置、初始或可编辑值；是否生效、何时下装须结合流程和 IED 能力。现场当前实时值。
运行值IED 服务读取、报告、GOOSE 或 SV 等运行数据。反映某时刻的服务数据；应连同 q、t、单位/倍率及测试状态判断。仅因 SCD 中有一个 Val 就已确定的实时状态。
枚举编码EnumType/EnumVal@ord 与相应 DA 类型定义。是某枚举文字与数值编码的类型定义；须沿 LN 类型、DOType、DAType 追溯其适用数据属性。任意 XML 属性枚举，或脱离类型上下文的业务状态解释。
三问法：这个值出现在哪个元素/数据属性？它是配置、服务读数还是类型定义？它的类型和 FC 是什么？答不清时，不要据数值猜业务含义。
