SCD 阅读工具箱

面对陌生节点时，用五个问题建立位置、引用、边界和值语义

陌生节点五问

无论看到 XML 元素、属性、DataSet 条目、通信参数或厂商扩展，先暂停“猜业务含义”，按以下五问记录证据：

问题怎样找输出什么
1. 位于哪个层级？从父元素一路向上记录路径，例如 SCL/IED/AccessPoint/Server/LDevice/LN0 或 SCL/Communication/SubNetwork/ConnectedAP/GSE。对象路径、所属 IED/AP/LD/LN 或网络子网；路径决定它属于设备模型、一次系统、通信、类型模板还是扩展。
2. 引用谁？检查 name、datSet、lnType、type、IED/LD/LN 标识、cbName 等属性。正向引用清单：它依赖的数据集、控制块、类型或设备实例。
3. 被谁引用？按名称、ID、LD/LN/DO/DA 路径和控制块名检索；对通信流从接收 ExtRef 反向追溯。反向引用清单：依赖它的 DataSet、控制块、LNode、ExtRef 或通信节点。
4. 标准还是扩展？确认元素/属性是否在项目要求版次的 SCL 通用模型中出现；检查命名空间和 Private。标记“标准语义”“工程配置”“厂商/工具扩展”或“待资料确认”。扩展不猜业务含义。
5. 配置值还是运行值？区分 DAI/Val、属性/参数、类型枚举，与 MMS/报告/GOOSE/SV 的运行服务数据。值的来源、数据属性、FC、类型链和时效性；不将配置或枚举编码当成现场实时状态。
最低可用结论：即使暂时无法理解业务，也应能写出它在何处、依赖谁、谁依赖它、标准/扩展边界和值的性质。

一分钟定位：先判断所属世界

若路径从这里开始优先阅读问题常见下一跳
Substation一次系统中的设备、间隔或功能位置是什么？VoltageLevel → Bay → ConductingEquipment / LNode
IED哪个 IED、AP、LD、LN 和数据对象提供功能？AccessPoint → Server → LDevice → LN0/LN → DO/DA
Communication哪个 IED/AP 接入哪里，哪个控制块怎样发布？SubNetwork → ConnectedAP → GSE / SMV → Address
DataTypeTemplates对象的 DO/DA 结构、枚举和类型来源是什么？LNodeType → DOType → DAType / EnumType
Private 或私有命名空间来源工具/厂商是谁，是否有资料可确认？保留原样、记录命名空间和上下文，转向项目/厂商资料。

引用追踪的可复用流程

A. 接收输入：ExtRef → 来源 IED/LD/LN/DO/DA → DataSet/FCDA → 控制块 → Communication/GSE 或 SMV
B. 发送控制块：GSEControl / SampledValueControl / ReportControl → datSet → FCDA → 接收 ExtRef 或客户端服务
C. 一次位置：Substation/Bay/LNode → iedName/ldInst/prefix/lnClass/lnInst → IED 内 LN → 数据与通信
D. 数值：DAI/Val 或运行读数 → LN@lnType → LNodeType → DOType → DAType / EnumType → FC、质量、时标
  1. 选定一个起点，只走一条链；不要同时在整份 SCD 中搜索所有同名对象。
  2. 每走一步记录当前节点、匹配属性、目标路径、是否唯一和证据来源。
  3. 出现多个匹配项时，补充 IED、AP、LD、prefix、LN 实例、FC 或控制块名缩小范围。
  4. 链条在标准对象处断开时检查工程版本/导出完整性；在 Private 处断开时标记待资料确认。
  5. 链条闭环后再判断工程合理性；引用存在不自动代表装置下装、网络转发或现场功能已验证。

值语义的三层判别

看到的值先问正确追踪避免的结论
DAI/Val这是工程初始值、可编辑设置还是设备配置？实例 → LN 类型 → DO/DA → FC → 设置/生效流程。“它就是当前现场运行值”。
EnumVal@ord它属于哪个 EnumType、哪个 DA？DA 类型引用 → EnumType → 文字/编码映射。“所有 0/1 或同名文字都有同一业务含义”。
报告/GOOSE/SV/MMS 读数这是哪项服务、何时产生、质量如何？服务上下文 → 数据引用 → q/t → 单位/倍率/测试状态。“有数值就表示数据有效或一次量正确”。
P@type 的文本该 P 参数的 type 是什么？按 MAC-Address、APPID、VLAN-ID、IP-SUBNET 等具体类型解释。“所有 P 都是 IP 地址或同一种网络参数”。

从阅读到行动的边界

当前证据可以得出的结论还不能得出的结论
XML 路径和引用可解析工程文件内部的部分结构与引用闭环。IED 已下装、服务已使能、网络已转发或现场功能已正确。
控制块/地址/ExtRef 可对应GOOSE 或 SV 的工程关联设计可被追踪。报文实际存在、订阅已生效、保护/控制业务已通过现场试验。
类型链和枚举可解析配置值或编码在该类型上下文中的含义。该值一定是运行实时值，或它的业务策略已在现场生效。
工具导入/校验通过该工具范围内的文件兼容或约束检查通过。跨厂商互操作性、网络部署或现场验收通过。
最后一问：若下一步会改变 IED、网络或一次设备状态，停止在“阅读工具箱”边界，转入项目授权的工程变更、试验和验收流程。
