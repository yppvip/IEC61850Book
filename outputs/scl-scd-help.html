如何阅读这份文档

SCD（Substation Configuration Description）是 SCL（Substation Configuration Language）的一种工程文件：它把一次系统、IED 配置、通信网络和数据类型模板放在同一 XML 中。XML 标签通常是英文；本页给出中文含义和“它与谁相连”。

阅读顺序建议：先看层级关系，再看 IED 内部的逻辑设备/逻辑节点，最后看数据集、FCDA、订阅关系和通信参数。看到厂商命名空间或 Private 时，应优先查该厂商文档。

标准语义 表示 IEC 通用概念；工程/私有 表示实例名或厂商扩展，不可直接推广为标准规则。

一张关系图

SCL ├─ Header 文件标识、版本和修改历史 ├─ Substation 一次系统模型 │ └─ VoltageLevel → Bay → ConductingEquipment / LNode ├─ Communication 网络与连接点配置 │ └─ SubNetwork → ConnectedAP → GSE / SMV / Address ├─ IED 一台智能电子设备 │ └─ AccessPoint → Server → LDevice → LN0 / LN │ ├─ DataSet → FCDA（引用的数据成员） │ ├─ ReportControl / GSEControl │ └─ Inputs → ExtRef（订阅的外部数据） └─ DataTypeTemplates 公共类型库：LNodeType / DOType / DAType / EnumType

同一业务对象会从不同视角出现：Substation 描述一次设备和间隔；IED 描述设备内部数据模型；Communication 描述它如何上网、发 GOOSE/采样值。LNode 将一次系统中的功能位置关联到某个 IED 内的逻辑节点。

SCL 根与 Header

元素/属性含义阅读要点
SCL整个文件的根元素。通常带 IEC SCL 默认命名空间；额外的 xmlns:xxx 往往是厂商扩展。
Header文件头。id 是文件标识；version/revision 是工程文件版本，不等同于 IEC 标准 Edition；toolID 指生成工具。
History/Hitem修改记录。when、who、what、why 适合审计，但内容质量取决于工程工具。
Private私有扩展容器。不是通用互操作语义；迁移、校验或自动处理时要保留，但不要据名称猜测功能。

Substation：一次系统视角

元素中文解释常见子元素/作用
Substation变电站/厂站模型的顶层。VoltageLevel、Function、Equipment。
VoltageLevel电压等级，例如 220 kV、110 kV。Bay（间隔）、Voltage 等；名称/描述应结合工程约定。
Bay间隔，如线路、主变、母联间隔。ConductingEquipment（断路器、刀闸、互感器等）和 LNode。
LNode一次系统中的逻辑节点引用。通过 iedName、ldInst、prefix、lnClass、lnInst 指向 IED 内的 LN/LN0。
易混淆点：Bay 不是逻辑节点，LN 也不是一次设备。前者表现一次系统结构，后者表现自动化功能数据模型；LNode 是二者的关联纽带。

Communication：通信网络视角

元素作用关键内容
SubNetwork通信子网，如站控网、过程层网。name、type；网络名是工程约定。
ConnectedAPIED 的某个 AccessPoint 接入某个子网。iedName、apName 必须能和 IED/AccessPoint 对上。
Address/P网络地址参数。P type="IP"、IP-SUBNET、IP-GATEWAY，或 MAC/VLAN 等；准确含义取决于通信服务。
GSEGOOSE（以及相关 GSE）发布端网络配置。ldInst、cbName 对应 IED 内的 GSEControl；还含 MAC、APPID、VLAN、最小/最大重发时间等。

IED：设备内部数据模型

元素含义关系
IED智能电子设备，如保护、测控、合并单元、网关。name 是工程中引用设备的关键标识；含 AccessPoint、Services、能力声明等。
AccessPointIED 对外提供服务的接入点。通常含 Server；其 name 需和 ConnectedAP@apName 一致。
ServerMMS 服务器逻辑容器。下含一个或多个 LDevice。
LDevice逻辑设备（LD）。inst 是实例标识；内部包含 LN0 与多个 LN。
LN0逻辑设备的零号逻辑节点。承载该 LD 的公共配置：数据集、报告控制块、GOOSE 控制块、设置组等常在这里。
LN逻辑节点，按功能组织数据。lnClass 是类别（如 XCBR、MMXU），inst 区分同类实例，prefix 是可选前缀。
DOI/SDI/DAI/Val实例化数据对象/子数据对象/数据属性及其配置值。它们是模板实例上的具体配置；不要误以为所有运行时值都应固定写在 SCD 中。

一个数据引用通常可概念化为：IED / AccessPoint / LDevice / LN / DO / DA。功能约束（FC）如 ST、MX、SP、SG 进一步说明该数据的用途。

DataSet 与 FCDA：发布或报告“数据清单”

DataSet 是一个数据条目列表。报告控制块、GOOSE 控制块等引用它，以决定要报告/发布哪些数据。每个 FCDA 是该清单中的一个功能约束数据属性引用。

FCDA 属性含义
ldInst逻辑设备实例。
prefix、lnClass、lnInst共同定位逻辑节点。
doName、daName定位数据对象和数据属性；daName 可进一步包含子属性路径。
fc功能约束。例如 ST（状态）、MX（测量）、SP（可设定参数）、SG（设置组）。它不是“数据类型”。
读法示例：<FCDA ldInst="LD0" lnClass="XCBR" lnInst="1" doName="Pos" daName="stVal" fc="ST"/>
表示“LD0 中 XCBR1 的 Pos 数据对象下 stVal 状态属性”，作为数据集的一员。实际实例名、是否存在该路径，以工程文件和类型模板为准。

Inputs 与 ExtRef：订阅外部数据

Inputs 位于接收方逻辑节点内，ExtRef 描述该节点要接收的外部数据引用。它常用于表达 GOOSE 或采样值等输入的工程绑定。

属性类别含义
本地输入点intAddr、desc 等，用于接收 IED 内部映射。
来源定位iedName、ldInst、lnClass、doName、daName 等，标识数据来自哪里。
服务/控制块关联可能含数据集、控制块、服务类型等信息；应与发送方 DataSet 和 GSEControl/报告配置交叉核对。

报告、GOOSE 与服务

元素用途阅读时核对
ReportControl定义报告控制块（RCB）：客户端读取或订阅的变化/周期报告。datSet、buffered、触发选项 TrgOps、可选字段 OptFields、RptEnabled。
GSEControl定义 GOOSE 发布控制块。name、datSet、appID；在 Communication 中查找同 LD、同控制块名的 GSE 配置。
ServicesIED 的服务能力声明。表示能力边界，不等于实际已经配置或启用的控制块。

DataTypeTemplates：类型库

模板作用典型关联
LNodeType逻辑节点的 DO 列表定义。LN/LN0@lnType 指向它。
DOType数据对象的 DA/SDO 结构定义。DO@type 指向它。
DAType结构化数据属性（BDA）的定义。DA@type 指向它。
EnumType枚举值及其数值编码。DA@bType="Enum" 时通常通过 type 引用。
排查某个 Val 的真正含义时，建议沿“实例 DAI → 对应 DA 定义 → DOType/DAType → EnumType”的路径追溯。不能只按显示文字猜枚举编码。

常见值：PIGO、ctlModel、on

值出现位置/性质解释
PIGO工程实例名 本实例中用于 Communication/GSE@ldInst。它是逻辑设备实例标识，应与 IED 内某个 LDevice@inst 及对应 GOOSE 控制块关联检查。不是 IEC 61850 规定的通用枚举，也不能从文字本身断定为“某种协议功能”。
sbo-with-enhanced-security标准控制模型枚举 通常作为控制对象的 ctlModel 数据属性值。“Select Before Operate（先选择、后执行）+ 增强安全”的控制模型。应用需先选择控制对象，再在有效时间内执行操作；具体交互字段与安全要求依赖 IED 支持能力及 IEC 61850-7-2/工程配置。
direct-with-normal-security标准控制模型枚举 ctlModel。直接执行、普通安全；不需要 SBO 选择步骤。是否允许遥控仍取决于闭锁、权限、模式和 IED 实现。
status-only标准控制模型枚举 ctlModel。只有状态，没有该对象的控制服务。
on需结合所在属性判断。常可表示布尔“真/启用/合”，但不是跨 SCL 所有元素通用的唯一含义。必须先看它属于 XML 属性、Val 文本还是某个 EnumType 成员；同样的 on 在不同数据类型中可能含义不同。

本工程实例观察：NB500SXS.scd

来源：用户提供的 NB500SXS.scd，读取时间 2026-07-17。以下是结构统计，不复制完整工程配置。

项目数量/观察解读
一次结构1 个 Substation；3 个 VoltageLevel；19 个 Bay可从一次系统视角定位间隔，再用 LNode 查找承载功能的 IED。
设备与接入点19 个 IED；47 个 ConnectedAP一台 IED 可以有多个接入点或连接到多个网络配置。
数据与订阅293 个 DataSet；10,944 个 FCDA；1,602 个 ExtRef数据集规模大；排错时应按某个控制块/订阅链路局部追踪，不宜人工通读。
服务控制块185 个 ReportControl；57 个 GSEControl报告和 GOOSE 都很常见；应交叉检查其数据集和通信侧配置。
类型模板546 个 LNodeType；95 个 DOType；25 个 DAType；22 个 EnumType数据语义应追溯类型模板，特别是枚举值。
FCDA 中出现的 FCMX、SG、SP、ST该数据集使用了测量、设置组、可设定参数和状态类数据。
实例里的 ctlModel 值0、direct-with-normal-security、sbo-with-enhanced-security、status-only文本值要和对应类型定义、IED 能力及项目控制策略一并确认；0 是否为特定枚举编码需沿模板追溯。

Edition 2 / 2.1 与文档边界

本页采用 Ed.2/Ed.2.1 的通用概念讲解。工程文件的 XML 命名空间仍可能显示较早的 SCL URI；不能仅靠命名空间字符串判断所有模型细节的 Edition。Ed.2.1 在建模和工程交换方面持续扩展，实际支持程度以 IED 供应商 ICD/CID、项目规范和一致性声明为准。

重要边界：厂商私有命名空间（本实例可见多个）和 Private 内容可能影响工具行为，但没有该厂商规范时，本帮助文档只把它们标为扩展，不给出推测性解释。涉及远方控制、联锁和安全策略时，应以项目设计和设备手册为准。

遇到一个陌生节点时的排查清单

  1. 先确认它位于 Substation、Communication、IED 还是 DataTypeTemplates。
  2. 看父节点和兄弟节点：SCL 的语义强依赖层级，而不是只依赖标签名。
  3. 把关键引用串起来：IED 名、AP 名、LD 实例、LN 类/实例、数据集名、控制块名。
  4. 若是 Val 或枚举，追溯到模板；若是文本实例名，搜索它被哪些引用使用。
  5. 若带私有前缀或位于 Private，记录厂商和工具来源，向项目/厂商资料确认。

IED 类型：保护、测控、MU、网关

标准语义 IED 是“智能电子设备”的总称。设备名称和 IED@type 多为工程约定；应以逻辑节点、通信配置和设备资料判断职责。

设备通常功能优先查看
保护装置保护逻辑、跳闸/告警、联跳闭锁。PTOC、PDIS、PTRC；ExtRef、GSEControl
测控装置遥测、遥信、遥控和间隔自动化。MMXU、XCBR、CSWI、CILO
合并单元 MU
Merging Unit		把 CT/VT 或电子式互感器的电流、电压采样数字化，并以过程层采样值发布；也可能接收 GOOSE。ConnectedAP/SMV、SampledValueControl、DataSet；常见 TCTR、TVTR、MMXU
智能终端连接一次设备机构：开入开出、位置/操作、GOOSE 联锁和跳闸执行。XCBR、CSWI、GGIO 及 GOOSE 输入/输出
网关/远动机汇集站内信息，向调度或其他系统转送。Services、报告控制块；外部规约映射通常不是 IEC 通用 SCL 语义
MU 不是交换机。MU 是发布数字化采样值的现场智能设备；交换机只承载以太网报文。MU 的发布流需和保护/测控对该 SMV 的订阅关联对应。

SMV / SV：MU 发布的采样值

SV（Sampled Values）是过程层连续电气量传输服务；SMV 常在 SCL 标签和工程口语中指采样值多播配置。典型链路：互感器二次量 → MU 采样/数字化 → 以太网 SV 报文 → 保护或测控接收。

位置元素/属性阅读含义
IED 内发布定义SampledValueControl、datSet、smvID定义采样值控制块及其数据集；可用属性会随 SCL 版本/工具变化。
网络配置ConnectedAP/SMV将发布控制块绑定到网络；核对 ldInst、cbName、Address。
二层地址MAC-Address、APPID、VLAN-ID、VLAN-PRIORITY标识以太网多播流并隔离/分级。数值是工程配置，不是任意复用的标准默认值。
接收关联Inputs/ExtRef接收逻辑节点引用发送侧数据；须和发送数据集、控制块、通信地址一起核对。
SV 不等于 GOOSE。SV 是高频连续采样量流；GOOSE 是面向状态/事件的快速多播消息。两者可使用 MAC、VLAN、APPID，但应用语义和检查方法不同。

GOOSE：快速事件与联锁消息

GOOSE（Generic Object Oriented Substation Event）常用于跳闸、闭锁、位置、告警和联锁。发送端 GSEControl 引用 DataSet；通信侧 GSE 赋予网络地址；接收端通过 ExtRef 建立订阅。

发送 IED：LN0/GSEControl ──datSet──▶ DataSet/FCDA │ └─ Communication：ConnectedAP/GSE ── MAC、APPID、VLAN、MinTime/MaxTime │ 接收 IED：LN/Inputs/ExtRef ◀──────────────────────┘
项目含义
appID 与 APPID前者常见于控制块，后者常在网络 Address/P。工具会关联它们，但不能只按名字断言所有版本的映射规则。
MinTime / MaxTime与 GOOSE 重发节奏有关，不是网络延迟保证值；性能应按项目规范验证。
ExtRef@serviceType若存在，可帮助识别输入服务类型；仍应核对完整来源引用和发送控制块。

常见逻辑节点（LN）速查

逻辑节点类别以 lnClass 表示功能，不直接代表一台物理设备。同一装置会有多个 LN；相同 LN 可由 prefix 和 inst 区分。

LN 类常见工程含义
LLN0 / LPHD逻辑设备公共节点 / 物理设备健康、名称等。
MMXU三相测量：电流、电压、功率、频率等。
XCBR / XSWI断路器 / 开关设备的位置、状态和控制对象。
CSWI / CILO开关控制 / 闭锁逻辑。
TCTR / TVTR电流互感器 / 电压互感器模型；过程层工程中常与 MU 数据关联。
PTOC / PDIS / PTRC过流保护 / 距离保护 / 跳闸条件。
GGIO通用 I/O；语义必须结合 desc、DO 名称和厂商资料，不可只凭 GGIO 猜测。
