# SCL / SCD 中文帮助文档

## 当前目标

生成可离线打开的静态 HTML，帮助不了解 IEC 61850 的读者理解变电站工程常见 SCD（SCL）结构、关键字、层级关系和常见约定值。第一阶段优先覆盖实际常用部分；文档使用中文，保留 XML 英文标签/枚举值。

## 依据与范围

- 参考口径：IEC 61850 Edition 2 / 2.1 的通用 SCL 实践。对 Edition 2.1 新增/变化之处作提示；不把厂商私有扩展当作标准定义。
- 工程实例（只读）：`E:\Document\项目\微机五防\全站SCD-工具版本14706\NB500SXS.scd`。
- 暂不处理大型实例：`E:\Document\项目\微机五防\HB-BD-XiongAn-220kV-JuCun-Stat-210419.scd`。
- 本阶段聚焦：`SCL`、`Header`、`Substation`、`Communication`、`IED`、`AccessPoint`、`Server`、`LDevice`、`LN0/LN`、`DataSet`、`FCDA`、`Inputs/ExtRef`、报告、GOOSE、`DataTypeTemplates`，以及常见控制/功能约定值。
- 不承诺逐条复述受版权保护的 IEC 标准；输出为工程学习用解释、关系图、示例和版本提示。

## 交付物

- `outputs/scl-scd-help.html`：单文件、无需联网、可在浏览器直接打开的帮助文档；包含目录、站内筛选和实例统计。

## 执行步骤

1. 读取小型 SCD 的根信息、常见元素和枚举用法，区分标准命名空间与私有扩展。
2. 编写新手导向的术语解释、层级关系、常用属性和值域说明。
3. 将实例中的统计和摘录加入 HTML；避免暴露无关工程配置全文。
4. 用浏览器/文本检查验证 HTML 的链接、筛选和 UTF-8 显示。后续可按用户提供的实际问题或大型 SCD 扩充词条。

## 写作约束

- “标准语义”“工程/厂商扩展”“示例观察”必须明确区分。
- 遇到 `Private` 或非 IEC 命名空间元素，只说明其扩展性质；没有厂商文档时不猜测业务含义。
- 约定值须说明它出现在哪个属性中，并避免将数据属性值与 XML 属性枚举混淆。
